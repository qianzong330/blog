权限概述

系统中提供了很多功能，并不是每个人登录系统之后，可以操作的功能是完全相同的。需要对系统的功能的访问进行控制。在我们的系统中，通过系统的菜单和按钮展示系统的功能，菜单和按钮就是某个功能的访问入口。

认证：系统提供的用于识别用户身份的功能（通常就是登录功能）—–你是谁？？？？
授权：系统中提供很多功能，需要为不同的用户根据其对应的权限，授予用户操作某些
功能的能力——你能做什么？？？

常见的权限控制的方式

url拦截进行权限控制

编写拦截器或者过滤器，在拦截器或过滤器中进行判断，当前登录用户是否具有访问当前功能的权限，如果有，方形，如果没有，跳转到权限不足的提示页面

方法注解方式权限控制

在相应的Action方法上添加@RequiredsPermmisons(“code”)的注解，如果某方法加了注解，就会为当前action创建代理对象，在代理对象中判断当前用户时候具有注解上所需要的权限，如果有，就通过反射调用action的相关方法，如果没有，跑出异常，阻止调用

Apache shiro框架

官方地址：Shiro.apache.org

Apache shiro是一个强大而灵活的开源安全框架，它能够干净利落的处理身份认证、授权、企业回话管理和加密。
以下是你可以用Apache Shiro所做的事情
1.验证用户
2，对用户执行访问控制（如判断用户是否拥有admin角色，是否拥有访问的权限）等

Shiro提供的权限控制的方式

1.url拦截进行权限控制–框架提供了很多过滤器进行url拦截
2.方法注解进行权限控制
3.页面标签权限控制–框架提供了一套标签，用于根据权限展示或隐藏按钮或菜单
4.代码级别权限控制–框架提供API,使用变成方式进行权限控制（不建议）

Shiro执行流程

Application Code —> Subject（当前用户）—> Shiro SecurityManager（安全管理器）—> Realm(安全数据桥，类似于Dao,负责访问安全数据)

快速入门 — 在项目中使用shiro

第一步：导入shiro的jar到项目中

第二步：在web.xml中配置一个过滤器代理对象，在项目启动时到spring工厂中加载一个和当前过滤器同名的bean对象

第三步：在spring配置文件中配置一个名称为shiroFilter的bean

第四步：修改UserAction的login登录访问，使用shiro框架提供的方式进行认证操作

第五步：自定义一个Realm，进行认证和授权操作

第六步：在spring配置文件中注册上面的realm，并注入给安全管理器
